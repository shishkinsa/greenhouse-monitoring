# Требования — Система мониторинга тепличного хозяйства

В этой папке собрана спецификация требований к информационной системе мониторинга тепличного хозяйства: веб-приложение для сотрудников и инженеров компании, просмотр видеопотока с камер в реальном времени с наложением показаний датчиков, поиск событий по критериям и просмотр архива видео и динамики датчиков. Масштаб: свыше 1000 теплиц по стране, около 1000 сотрудников и около 50 инженеров.

---

## Исходный документ

**[00 requirement.md](00%20requirement.md)** — исходный текст требований заказчика/стейкхолдеров. На его основе выведены бизнес-цели, пользовательские истории, бизнес-правила, функциональные (FR) и нефункциональные (NFR) требования.

## Связи между документами

Исходные требования ([00 requirement.md](00%20requirement.md)) разнесены по уровням:

- **Бизнес-уровень:** цели (BR-01 … BR-04), пользовательские истории (US-01, US-02), бизнес-правила (BR-R01 … BR-R07).
- **Функциональные требования** (FR-01 … FR-03) реализуют сценарии из пользовательских историй.
- **Нефункциональные требования** (NFR-01 … NFR-04) задают ограничения по задержке, отклику, доступности и масштабу.

Количественные условия из бизнес-правил BR-R05, BR-R06, BR-R07 детализированы в NFR-01, NFR-02 и NFR-03 соответственно.
