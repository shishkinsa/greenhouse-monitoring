## US-01: Просмотр видеопотока в реальном времени (Сотрудник)

### Метаданные

| Атрибут | Значение |
|---------|----------|
| **ID** | US-01 |
| **Тип** | Бизнес |
| **Статус** | Утверждено |
| **Приоритет** | Критический |
| **Автор** | Команда проекта |
| **Владелец** | Команда проекта |
| **Создано** | 2025-03-14 |
| **Обновлено** | 2025-03-14 |
| **Версия** | 1.0 |

### Описание

Как сотрудник компании (~1000 человек), я хочу заходить в веб-приложение, выбирать конкретную теплицу и видеокамеру и смотреть в реальном времени видеопоток с наложением в нижнем правом углу актуальных показаний датчиков (температура, влажность, кислотность почвы), чтобы оперативно контролировать состояние теплицы.

### Детальное описание

Сотрудник после входа в систему выбирает теплицу из списка (или по иерархии регион/объект), затем выбирает одну из видеокамер этой теплицы (до 5 камер на теплицу). На экране отображается видеопоток по RTSP с выбранной камеры; в нижнем правом углу — значения трёх датчиков, обновляемые в реальном времени по данным MQTT. Задержка отображения данных и видео не должна превышать 30 секунд.

### Критерии приёмки

```gherkin
Функционал: Просмотр видеопотока в реальном времени
  История: Сотрудник смотрит видео с камеры теплицы с показаниями датчиков в углу экрана

  Сценарий 1: Успешный просмотр видеопотока с данными датчиков
    Допустим сотрудник авторизован в веб-приложении
    И теплица и камера доступны и передают данные
    Когда сотрудник выбирает теплицу и видеокамеру
    Тогда отображается видеопоток с выбранной камеры
    И в нижнем правом углу отображаются температура, влажность и кислотность почвы
    И задержка отображения данных и видео не превышает 30 секунд

  Сценарий 2: Недоступность камеры или контроллера
    Допустим сотрудник авторизован в веб-приложении
    И выбранная камера или контроллер теплицы недоступны
    Когда сотрудник выбирает теплицу и видеокамеру
    Тогда отображается сообщение об ошибке или заглушка
    И пользователь информирован о недоступности источника данных
```

## US-02: Поиск событий по критериям и просмотр архива (Инженер)

### Метаданные

| Атрибут | Значение |
|---------|----------|
| **ID** | US-02 |
| **Тип** | Бизнес |
| **Статус** | Утверждено |
| **Приоритет** | Критический |
| **Автор** | Команда проекта |
| **Владелец** | Команда проекта |
| **Создано** | 2025-03-14 |
| **Обновлено** | 2025-03-14 |
| **Версия** | 1.0 |

### Описание

Как инженер компании (~50 человек), я хочу набирать арифметико-логические выражения по показателям датчиков для задания критерия отбора событий, выбирать теплицы и временной интервал, а затем просматривать архивный видеопоток с камер и динамику изменения показателей датчиков по отобранным событиям, чтобы анализировать произошедшие инциденты и отклонения.

### Детальное описание

Инженер после входа в систему вводит критерий отбора событий в виде выражения над показателями датчиков (например, температура > 30 AND влажность < 60), выбирает теплицы (одну или несколько) и временной интервал. Система возвращает список событий, удовлетворяющих критерию. Время отклика интерфейса на запрос не должно превышать 10 секунд. Для выбранных событий инженер может просмотреть архивное видео с камер соответствующей теплицы и графики динамики показателей датчиков.

### Критерии приёмки

```gherkin
Функционал: Поиск событий и просмотр архива
  История: Инженер задаёт критерий по датчикам и просматривает архив видео и динамику датчиков по событиям

  Сценарий 1: Успешный поиск событий и просмотр архива
    Допустим инженер авторизован в веб-приложении
    И в выбранных теплицах и интервале есть данные датчиков и архив видео
    Когда инженер вводит арифметико-логическое выражение по показателям датчиков
    И выбирает теплицы и временной интервал
    И отправляет запрос на поиск событий
    Тогда список событий отображается не более чем через 10 секунд
    И инженер может открыть архивный видеопоток с камер выбранной теплицы по событию
    И инженер может просмотреть динамику изменения показателей датчиков по событию

  Сценарий 2: Пустой результат или ошибка запроса
    Допустим инженер авторизован в веб-приложении
    Когда инженер вводит критерий и выбирает теплицы и интервал
    И событий по критерию не найдено или произошла ошибка
    Тогда отображается сообщение о пустом результате или об ошибке
    И интерфейс остаётся отзывчивым (не более 10 секунд до ответа)
```
