# Инструкции для AI Cursor

## Как использовать документацию

### 1. Всегда начинай с контекста
```yaml
before_any_task:
  - "Определи текущий контекст (backend/frontend/architecture)"
  - "Найди соответствующие документы в /docs"
```

### 2. Требования к системе должны быть всегда актуальна

```yaml
requirements:
  - "Требования к документации должны быть оформлены в соответствии с (docs/requirements/template.md)"
  - "Требования к системе должны быть всегда актуальна. Изменения вносятся сначало в требования потом в архитектуру и уже на последнем этапе в код"
```

### 3. Требования к архитектуре

```yaml
requirements:
  - "Разрабатываемая архитектура системы должна соответствовать расчетам (docs\architecture\сalc_architecture.md)"
```
