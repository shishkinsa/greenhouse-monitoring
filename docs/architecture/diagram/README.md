## Структура папок

Каталог диаграмм организован по типам представлений архитектуры (соответствует диапазонам номеров в [принципах именования](#номера-по-категориям)):

```text
docs/architecture/diagram/
├── README.md              # Этот файл — описание раздела и правил
├── context/               # 01–09: контекст системы, внешние акторы и системы
├── containers/            # 01–09: контейнеры (сервисы, БД, очереди)
├── components/            # 10–19: компоненты внутри контейнеров
├── data/                  # 20–29: схемы данных, ER, потоки данных
├── infrastructure/        # 30–39: серверы, сети, облако, деплой
├── behavior/              # 40–49: последовательности, состояния, сценарии
├── business/              # 50–59: бизнес-процессы и потоки работ
├── security/              # 60–69: угрозы, аутентификация, авторизация
├── integration/           # 70–79: интеграции с внешними системами
└── legacy/                # 80–99: устаревшие или временные диаграммы
```

| Папка | Назначение |
|-------|------------|
| `context/` | Контекстная диаграмма: кто взаимодействует с системой, внешние системы |
| `containers/` | Высокоуровневые сервисы, БД, очереди — границы системы |
| `components/` | Внутреннее устройство сервисов для разработчиков |
| `data/` | Схемы БД, связи между таблицами, потоки данных |
| `infrastructure/` | Серверы, сети, облачные ресурсы, деплой |
| `behavior/` | Диаграммы последовательностей, состояний, сценариев |
| `business/` | Бизнес-логика и потоки работ |
| `security/` | Угрозы, потоки аутентификации и авторизации |
| `integration/` | Связи с внешними API и системами |
| `legacy/` | Устаревшие или временные диаграммы, миграции |

### Примеры
| Файл | Описание |

## 🗺️ Типы диаграмм

| Категория | Что показывают | Примеры использования |
|-----------|---------------|----------------------|
| **Контекст** | Кто взаимодействует с системой, внешние системы | Для новых членов команды, стейкхолдеров |
| **Контейнеры** | Высокоуровневые сервисы, БД, очереди | Для понимания границ сервисов |
| **Компоненты** | Внутреннее устройство каждого сервиса | Для разработчиков сервиса |
| **Данные** | Схемы БД, связи между таблицами | Для DBA, бэкенд-разработчиков |
| **Инфраструктура** | Серверы, сети, облачные ресурсы | Для DevOps, SRE |
| **Поведение** | Как система работает во времени | Для отладки, оптимизации |
| **Бизнес-процессы** | Бизнес-логика и потоки работ | Для аналитиков, Product Owners |
| **Безопасность** | Угрозы, потоки аутентификации | Для security-инженеров |
| **Интеграции** | Связи с внешними системами | Для интеграторов |

## 🔄 Генерация изображений

### Локальная генерация

```bash
# Из корня проекта
make diagrams

# Или напрямую скриптом
./scripts/generate-diagrams.sh
```

### Результат

PNG файлы появятся в public/ с сохранением структуры:

```text
public/
├── context/
│   └── 01-system-context.png
├── data/
│   └── schema/
│       └── 01-users.png
├── behavior/
│   └── sequence/
│       └── 01-auth-flow.png
└── ...
```

### Автоматическая генерация при изменениях

```bash
# Установить fswatch
brew install fswatch

# Запустить отслеживание
fswatch -o architecture/diagram | xargs -n1 ./scripts/generate-diagrams.sh
```
